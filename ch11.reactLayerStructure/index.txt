react app의 복잡성이 높아지면서 코드를 관리하기 쉽지 않네
계층구조를 BE뿐 아니라 FE에도 도입해보자

-----------------------------------------------------------------------------------

아래의 문제를 해결할 수 있다.

1. 관심사 분리 : 계층별 책임이 분리되어 검색하고 찾기 수월
2. 높은 재사용성 : 비즈니스 로직과 데이터 모델을 app에서 사용하기 쉬워짐
3. 테스트 용이성 : 단위테스트 / 통합테스트를 작성하기 쉬워짐
4. 유지보수성 : 기능 추가가 쉬워짐

-----------------------------------------------------------------------------------

소규모 프로젝트의 경우 몇개의 컴포넌트에 로직 다 써도 유지보수 쉬움 ㅋ
코드베이스 커지면 유지보수가 어렵네에..

소규모면 극단적으로 단일 컴포넌트 하나로 다한다.ㅋㅋ
(fetch / state / domain / render 다 담당)
관리가 쉽지는 않을듯 ㅋㅋ

이걸 여러 컴포넌트로 분리하면 각 컴포넌트의 기능 구현에만 집중할 수 있어서 좀더 나아짐
역할이 나눠진건 좋아졌지만, 여전히 뷰가 많은 역할 (로직/상태)를 가지고 있다.

이걸 hooks로 분리해서 상태와 로직은 분리시키자. 
컴포넌트는 렌더나 하쇼 (Pure Component)

추가적으로 도메인 로직을 별도로 분리하자.
  컨테이너 컴포넌트 - hook(상태관리) - domain(로직)

UI와 관련없고 데이터 출처와도 관련이 없는 객체 집합을 정의할 수 있다. (도메인)
API 호출 / 로그인 / 결제 등 인터페이스를 맞춰주는 레이어도 있다.

https://martinfowler.com/bliki/PresentationDomainDataLayering.html
