리액트 프로젝트를 구조화하는 전략에 대해 알아보자
잘 갖춰진 구조는 유지보수하기 쉬워 프로젝트 성공에 긍정적 영향을 이끌수 있다.

------------------------------------------------------------------------------------

구조화되지 않은 프로젝트의 문제점
- 프로젝트가 커질수록 구조화되지 않은 문제점이 드러난다.
- 코드베이스를 효과적으로 관리하고 유지보수하기 어렵다.

1. 무질서 창궐 : 무언가 찾기가 쉽지 않음
2. 낮은 재사용성 : 무엇이 재사용가능한지 알기 어려움
3. 협업의 어려움 : 서로의 코드를 이해하고 탐색하는것이 어려움

------------------------------------------------------------------------------------

프로젝트는 아래의 요소로 구성되며 이를 일관되게 정리해야한다.

1. 소스코드
  - 공통 컴포넌트
  - 특정 기능만을 위한 컴포넌트 
  - 스타일 정의
  - hooks / utils
2. asset: 폰트 / 이미지 / 비디오같은 정적 파일
3. config: package / build / deploy / linting 등을 위한 설정
